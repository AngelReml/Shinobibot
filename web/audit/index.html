How it works

Your agent stays where it is. We run tasks against it, verify the output deterministically, and write each verdict into an immutable hash chain.

1

OpenAI-compatible HTTP endpoint, Hermes/Shinobi runtime, or anything that speaks JSON. We never touch your model weights.

2

AuditGravity kernel

Runs shinobi-bench public tasks plus your private bench. Each call: deterministic verifier → PASS / FAIL.

3

Every result lands in a SHA-256 chain (genesis → entry_n). You can recompute the hash locally — no trust in us required.

Compatibility

If your agent talks HTTP or speaks an OpenAI-compatible API, we audit it.

  • Hermes — native runtime support via shinobi import hermes
  • Shinobi — native; ships with the agentskills.io catalog
  • Claude Code / Claude API — auditable via OpenAI-compat proxy
  • OpenAI Codex / ChatGPT API — same proxy
  • OpenAI Operator — webhook + result attestation
  • Custom agents (Bedrock, Azure OpenAI, on-prem llama.cpp, …) — JSON HTTP contract, 30 LoC
