Calibration period (transparency)

AuditGravity needs a baseline before scoring is meaningful. The first 20 occurrences of each action type per agent are marked calibrating — the score is forced to 0 while we learn the agent's normal behavior. Action 21 onward is scored against that baseline. The /profile endpoint exposes { status: "calibrating" | "active", calibration_progress: "60%" } so you always know which mode each agent is in. No festival of false positives at deployment.

How it works

Your agent emits an action. We score it 0–10 against a learned baseline. We block, warn, or allow — and write the score to a hash chain.

Compatibility

If your agent talks HTTP or speaks an OpenAI-compatible API, we audit it.

  • Hermes — native runtime support via shinobi import hermes
  • Shinobi — native; ships with the agentskills.io catalog
  • Claude Code / Claude API — auditable via OpenAI-compat proxy
  • OpenAI Codex / ChatGPT API — same proxy
  • OpenAI Operator — webhook + result attestation
  • Custom agents (Bedrock, Azure OpenAI, on-prem llama.cpp, …) — JSON HTTP contract, 30 LoC
