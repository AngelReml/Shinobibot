# Sesión de Desarrollo: Bloque 13 — Setup Comercial (zapweave.com)
**Fecha:** 2026-05-04

## Resumen de la Operación
Se construyó el sitio comercial completo y la documentación pública del
proyecto. Todo es estático (HTML + CSS), sin gastar dinero, listo para
desplegar en GitHub Pages en `zapweave.com`. Nicho explícito durante
todo el copy: **YouTubers hispanos que adaptan formatos de canales en
inglés**. Ningún testimonio inventado.

## 1. Sitio web (`web/`)
Cuatro páginas estáticas con un único `styles.css` compartido. Tema
oscuro minimalista inspirado en Linear / Vercel.

| Archivo | Propósito |
|---------|-----------|
| `web/index.html` | Landing: titular, "cómo funciona" en 3 pasos, ventajas, lista de espera (Formspree placeholder), footer. |
| `web/precios.html` | Shinobi gratis para siempre · OpenGravity Founders 9 €/mes (500 plazas, precio bloqueado) · OpenGravity Pro 19 €/mes con 7 días de prueba. |
| `web/catalog.html` | Marketplace de skills con 6 placeholders "Próximamente". |
| `web/docs.html` | Instalar exe, pedir API key, comandos CLI, FAQ con 5 preguntas (privacidad, LLMs, créditos, cancelar, datos al cerrar cuenta). |
| `web/styles.css` | Sistema de tokens (`--bg`, `--accent`, `--text-dim`, etc.), responsive a 720/860 px. |

### Decisiones de copy
- Titular: "Para YouTubers hispanos que adaptan **formatos en inglés** a
  su canal en español." Nicho explícito, sin disimular.
- Lista de espera: `https://formspree.io/f/REPLACE_FORMSPREE_ID` —
  **TODO**: el usuario crea el form gratuito y reemplaza el ID. Mientras
  tanto el footer ofrece `mailto:hola@zapweave.com` como respaldo.
- Social proof: bloque comentado en `index.html` con un TODO claro.
  No hay testimonios inventados ni números falsos de usuarios.

## 2. README maestro (`README.md`)
Resumen del producto, distinción Shinobi (local) vs. OpenGravity (cloud
de pago), tabla del estado **B0–B13 cerrados**, instrucciones de setup
dev (`npm install`, `.env`, `npm run dev`), build a `.exe` (`build_sea.mjs`,
`rebuild.cmd`, `rebuild_test.cmd`), estructura de carpetas y link a
`zapweave.com`.

## 3. Email de bienvenida (`docs/emails/welcome.md`)
Plantilla con:
- Tono directo, sin marketing.
- Agradecimiento sincero al beta tester.
- Variables para sustituir: `{{nombre}}`, `{{SHINOBI_API_KEY}}`,
  `{{download_url}}`.
- Expectativas claras: producto temprano, fallos esperados.
- Founders: precio 9 €/mes bloqueado **mientras la suscripción siga
  activa** (si cancela y vuelve, no se garantiza recuperarlo).
- Promesas explícitas de lo que NO se hará: no newsletter automática,
  no venta de email, no testimonios inventados.

## 4. Discord (`docs/comunidad.md`)
**No se crea desde código.** Documento con pasos manuales para que el
usuario cree el server (10 minutos):
- Canales: `#welcome`, `#anuncios` (admin-only), `#bugs`, `#ideas`,
  `#showcase`, `#charla` (off-topic opcional).
- Roles: `Founders`, `Beta`, `Mod`, `Bot`.
- Pins listos para copiar: bienvenida + reglas, plantillas de bug y
  de idea.
- Permisos por canal, verificación, configuración de notificaciones.
- Mantenimiento mensual y "qué hacer cuando tengas el invite".

## 7. Pendientes para el usuario
1. Crear el form gratuito en Formspree y reemplazar `REPLACE_FORMSPREE_ID`
   en `web/index.html`.
2. Configurar el repo de GitHub Pages apuntando a `web/` (o subir el
   contenido a un repo `zapweave.github.io`).
3. Crear el dominio `zapweave.com` y apuntar DNS al CNAME de GitHub Pages.
4. Crear el Discord siguiendo `docs/comunidad.md` y devolver el invite
   para añadirlo al README, landing y email.
5. Crear un email transaccional `hola@zapweave.com` (Google Workspace,
   Fastmail o forwarding gratuito de Cloudflare).
