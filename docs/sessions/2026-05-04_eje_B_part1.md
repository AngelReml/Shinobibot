## Estructura por skill

```
skills/desktop/<name>/
├── SKILL.md           # frontmatter agentskills.io + body con parámetros / edge cases
└── scripts/
    ├── skill.mjs      # entry Node, llama a registerTool
    └── <helper>.{ps1,jsx}  # automatización vía COM o ExtendScript
```

Las skills son **bundles agentskills.io directos** (no pasan por SkillGenerator LLM): cada `SKILL.md` está hand-tuned para enumerar parámetros, output, edge cases y consideraciones de seguridad. Cada `skill.mjs` registra una tool con `parameters` JSON-Schema y `execute(args)` que devuelve `ToolResult`.

## Decisiones técnicas

1. **COM para Excel y Outlook, ExtendScript para Adobe**: vía estándar Microsoft/Adobe, no requiere libs externas, soporta múltiples versiones de Office/Adobe.
2. **obs-websocket v5 y CDP **inline**, sin deps**: aprovecho `globalThis.WebSocket` nativo de Node 22+. Evita bloat de `obs-websocket-js`/`chrome-remote-interface`.
3. **Templates `.jsx` con replace**: simple, predecible, depurable. Resultado vuelve por archivo `result.json` en `%TEMP%`.
4. **`findPremiereExe()` / `findPhotoshopExe()` / `findChromeExe()`** con whitelisting de paths estándar Adobe + Google. Nuevas versiones se añaden sin tocar el resto.
5. **Password redaction**: `desktop-outlook-send-email` nunca loguea `body`; `desktop-chrome-login-and-action` redacta `text` cuando el selector contiene `[type=password]`.
6. **Safety**: SKILL.md de Outlook y Chrome documentan explícitamente "el agente debe confirmar con el usuario antes de invocar".
7. **No deps nuevas**: ningún `package.json` modificado; todo usa stdlib + el `tool_registry` existente.
