## Decisiones técnicas

1. **`ObsClient` extraído a `src/skills_runtime/`**: misma clase usada por B4.4 (setup-scene), H1 (record-self) y H2 (stop-and-save). Dependencia 0 (usa `globalThis.WebSocket` de Node 22+).
2. **Auto-launch OBS**: H1 detecta `obs64.exe` en paths estándar y lo lanza con `--minimize-to-tray` + reintenta connect cada 500ms hasta 12s. Si el usuario no tiene OBS instalado, el error es claro.
3. **`waitForFileReady`**: H2 espera el handle libre con `openSync(path,'a')+closeSync()` antes de devolver, evitando leer un MP4 que OBS aún no ha cerrado.
4. **Demo runner usa el verifier público**: primero busca el sibling `OpenGravity/src/benchmark/shinobi/verifiers.mjs`, luego cae a `raw.githubusercontent.com/.../verifiers/index.mjs` con caché en `demos/.cache/`. Demo es **runnable desde clone limpio**.
5. **Modo `--no-record`** (no-OBS): permite tests CI deterministas sin requerir OBS instalado. La sesión se imprime a stdout + transcript file en `demos/`.
6. **Subset narrado de 7 tareas** en `full-self-improve`: T11/T13/T16/T17/T22/T25/T28. Suficiente para una demo de 30s–2min, no espera el bench completo.
7. **Subtítulos Whisper**: documentados como TODO en `manual_actions.md`. Se enchufan cuando se aborde B3 (voice mode) — la API ya admite hook por callback en `demo_runner`.
