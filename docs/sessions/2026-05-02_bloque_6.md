# Sesión de Desarrollo: Bloque 6 — Modo Agente Residente en Shinobi
**Fecha:** 2026-05-02

## Componentes Creados

### `src/persistence/missions_recurrent.ts` — MissionsStore
- Persistencia SQLite (`%APPDATA%/Shinobi/missions.db`) con WAL mode.
- Tabla `missions_recurrent`: CRUD completo para misiones recurrentes.
- Tabla `mission_logs`: historial de ejecuciones con output y errores.
- **Circuit breaker**: tras 3 fallos consecutivos, la misión se desactiva automáticamente.
- `getDueMissions()`: filtra misiones habilitadas cuyo `cron_seconds` ha transcurrido desde `last_run_at`.
- `resetFailures()`: restablece el contador para reactivar una misión bloqueada.

### `src/runtime/resident_loop.ts` — ResidentLoop
- Timer basado en `setTimeout` con tick cada 30 segundos.
- En cada tick, ejecuta todas las misiones "due" vía `ShinobiOrchestrator.process()`.
- Registra éxito/fallo en `MissionsStore`.
- Si una misión alcanza 3 fallos consecutivos, envía notificación vía `Notifier`.

### `src/notifications/notifier.ts` — Notifier
- Sistema de notificaciones configurable.
- Sin workflow configurado: imprime a consola (fallback seguro).
- Con workflow configurado: invoca `OpenGravityClient.invokeWorkflow()` para despachar alertas vía n8n (Slack, email, etc.).

## Comandos CLI

### `/resident`
| Comando | Descripción |
|---------|-------------|
| `/resident start` | Arranca el loop residente |
| `/resident stop` | Detiene el loop |
| `/resident status` | Muestra estado del loop y misiones |
| `/resident add "nombre" <segundos> <prompt>` | Crea misión recurrente |
| `/resident enable <id>` | Habilita una misión |
| `/resident disable <id>` | Deshabilita una misión |
| `/resident delete <id>` | Elimina una misión |
| `/resident reset <id>` | Restablece contador de fallos |
| `/resident logs <id>` | Muestra últimos 5 logs |

### `/notify`
| Comando | Descripción |
|---------|-------------|
| `/notify set <workflow_id>` | Configura workflow de n8n para alertas |
| `/notify unset` | Deshabilita notificaciones externas |
| `/notify test` | Envía notificación de prueba |

## Notas de Diseño
- El loop NO está instalado como servicio Windows. Se arranca/detiene manualmente con `/resident start|stop` dentro del CLI.
- El tick interval (30s) es suficiente para misiones con cron ≥ 60s. Para misiones más frecuentes, se podría reducir.
- Las misiones bloqueadas por circuit breaker se reactivan con `/resident reset <id>`.
