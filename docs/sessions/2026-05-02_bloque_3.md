# Sesión de Desarrollo: Bloque 3 - Memoria estilo OpenClaw
**Fecha:** 2026-05-02

## Decisión Arquitectónica
Tras la inspección profunda de la base de código de OpenClaw, se decidió **NO clonar OpenClaw**. Su arquitectura es un monorepo complejo basado en un sistema de plugins muy acoplado (`plugin-sdk`, `manager-*.ts`, etc.) que es imposible de extraer limpiamente hacia Shinobi.

En su lugar, **se ha reescrito el patrón principal** directamente en Shinobi con Typescript, basándonos en:
- Markdown / Strings como fuente de datos de la memoria.
- `better-sqlite3` para persistencia robusta y local-first.
- Búsqueda semántica usando embeddings generados vía OpenRouter (usando llamadas al kernel en formato JSON), mitigando la falta de capacidades de vector nativas por ahora a través de cosine similarity iterativo local.
- *Short-term promotion* (actualización de métricas de acceso, tracking de logs de recall) e inyección automática en los `system` prompts (Context Builder).

## Archivos Creados/Modificados
1. `src/memory/types.ts`: Tipado unificado de los fragmentos de memoria, parámetros de recall, etc.
2. `src/memory/embedding_provider.ts`: Interfaz con OpenGravityClient que orquesta el fallback dinámico (pseudo-hash determinista) o embeddings reales vía LLM.
3. `src/memory/memory_store.ts`: El gestor de la base de datos local SQLite con tabla `memories`, logs de recall `recall_log`, heurísticas de validación y utilidades.
4. `src/coordinator/orchestrator.ts` (Modificado): Inicialización estática del memory-store y lógica de inyección de memoria automática al prompt `system`.
5. `scripts/shinobi.ts` (Modificado): CLI extendida con `/memory recall`, `/memory store`, `/memory stats` y `/memory forget`.
6. `test_b3.ts`: Script de validación de integración end-to-end.

## Limitaciones Conocidas
- **Embeddings vía LLM JSON (No Nativo)**: Actualmente dependemos del modelo generativo (OpenRouter/GLM) para emitir los arrays JSON simulando vectores, lo cual es ineficiente y muy lento a nivel computacional/económico. En un futuro cercano, se deberá considerar la instalación de un modelo `sentence-transformers` en local (como en OpenClaw, usando extensiones vec en C++ on-device).
- **SQLite Vec missing**: Por ahora, la similitud coseno se está calculando iterativamente con Typescript en RAM (`EmbeddingProvider.cosineSimilarity`), lo que no escalará si se tienen +10,000 memorias en producción.
