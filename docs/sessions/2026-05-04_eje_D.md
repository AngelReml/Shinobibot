## D4 — SDKs + webhook + proxy

### SDK Python (`sdks/python/auditgravity`)

- Stdlib-only (urllib). Min Python 3.10.
- API: `audit(agent, task, response, ...)` y `AuditClient(...)`.
- Empaquetado con hatchling (`pyproject.toml`).
- Publicación a PyPI = TODO manual.

### SDK Node (`sdks/node`)

- ESM puro, native `fetch` (Node 20+). Cero deps.
- API espejo del Python.
- Publicación a npm = TODO manual.

### Webhook receiver `POST /v1/audit/event`

`src/audit/event_log.ts` — log SHA-256 hash-chained, anchor genesis. Independiente de `forensic/recorder.ts` (sectores) y `benchmark/shinobi/ledger.ts` (bench runs); cada productor escribe en su propio chain.

`src/api/v1/audit_router.ts` monta:
- `POST /v1/audit/event` — recibe `{agent, task, response, verifier?, verifier_args?, metadata?}`. Si pasa `verifier`, ejecuta el verifier registrado (de la public registry de shinobi-bench) y devuelve `verdict` + `chain_entry_hash`.
- `GET /v1/audit/events/recent?limit=N` — tail.
- `GET /v1/audit/chain/verify` — re-hash y verifica integridad.

### OpenAI-compat proxy

`POST /v1/audit/openai/v1/chat/completions` — transparent passthrough a `UPSTREAM_OPENAI_BASE_URL` (default `https://api.openai.com`) con `UPSTREAM_OPENAI_KEY`. Tras el round-trip, anota `(last_user, assistant_reply)` como evento de audit con `source=proxy`.

Cliente B2B reconfigura `OPENAI_BASE_URL=https://kernel.zapweave.com/v1/audit/openai` en su SDK oficial y queda auditado sin tocar código.
