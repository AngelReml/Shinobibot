Distribución detallada:

- **5 puntos**: P-001, P-002, P-003, P-004, P-009, P-011, P-012 — schema sólido pero sin ejemplos ni auto-check.

**Patrones de mejora identificados (aplicables a múltiples prompts):**
1. **C1 → 2**: añadir nivel + especialización + frase de contexto operativo (ej. "auditando agentes LLM").
2. **C4 → 1-2**: incluir 1 ejemplo positivo extraído de evidencia real del repo + (cuando aplique) 1 contraejemplo.
3. **C5 → 1-2**: añadir auto-check de UNA línea, específico al riesgo del rol (invención de paths, cita sin línea, dissent suprimido, etc.).

Cada propuesta de reescritura es self-contained y lista para reemplazar el string original en el Paso 5. Todas mantienen el contrato externo (mismo schema JSON de salida) — el código consumidor no requiere cambios.
