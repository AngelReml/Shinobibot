# Sesión de Desarrollo: Bloque 9 — Operación de Apps de Escritorio sin API
**Fecha:** 2026-05-04

## Resumen de la Operación
Shinobi ahora puede operar aplicaciones de escritorio Windows que no exponen API,
mediante un loop **ver → pensar → actuar**. Se añadieron dos herramientas nativas
(`screen_observe`, `screen_act`), una utilidad de visión (OpenRouter / OpenAI),
una *kill switch* por ESC y filtros de seguridad para zonas prohibidas y acciones
destructivas.

## 1. Stack y Decisiones
- **Automatización screen+mouse+keyboard**: `@nut-tree-fork/nut-js` (fork mantenido,
  sin compilación nativa fallida en Node 24). No fue necesario fallback a robotjs.
- **Vision LLM**: OpenRouter `google/gemini-2.0-flash-001` por coste, con fallback
  a OpenAI `gpt-4o-mini` si `OPENROUTER_API_KEY` no está presente.
- **Kill switch**: PowerShell hijo que poll-ea `GetAsyncKeyState(VK_ESCAPE=0x1B)`
  cada 50 ms; emite `STOP` a stdout cuando ESC se mantiene >= 1000 ms. El proceso
  Node lo lee y aborta la siguiente acción.

## 2. Archivos Añadidos
- `src/utils/vision_client.ts` — wrapper REST a OpenRouter / OpenAI Vision.
- `src/utils/kill_switch.ts` — proceso PowerShell que vigila ESC.
- `src/utils/screen_safety.ts` — checks de zona prohibida, título de ventana y
  detección de intent destructivo.
- `src/tools/screen_observe.ts` — captura PNG con nut.js + descripción del LLM
  con bounding boxes en píxeles.
- `src/tools/screen_act.ts` — `click / double_click / right_click / move / type /
  press_key / hotkey / scroll`.
- `test_b9.ts` — E2E sobre Notepad.

`src/tools/index.ts` registra los dos nuevos tools.

## 3. Restricciones Implementadas (B9 obligatorias)
- **Zonas prohibidas**:
  - Banda inferior 40 px (taskbar / system tray).
  - Coordenadas fuera de `screen.width()/height()`.
  - Títulos de ventana que matchean `/user account control/i`,
    `/windows security/i`, `/administrator: /i`, `/system32/i`,
    `/windows\\system/i`, `/control panel/i`, `/windows update/i`.
- **Acciones destructivas** (gate por confirmación CLI):
  - `Alt+F4` (cerrar ventana, posible discard).
  - `Ctrl+W` (cerrar pestaña).
  - tecla `Delete` o `Shift+Delete`.
  - `type` con texto que contiene `rm -rf`, `del /s`, `format`.
  - El parámetro `force_confirm: true` salta el prompt cuando el caller ya confirmó.
- **Kill switch ESC**: ESC mantenido ≥ 1 s aborta la herramienta antes y después
  de cada acción.

## 5. Limitaciones Conocidas
- El detector de "ventana activa" depende de `nut.getActiveWindow()`; en algunos
  casos puede devolver `null` y se asume permiso (se sigue con los otros checks).
- El gate destructivo no entra en TTYs no interactivas: en ese modo se rechaza
  por defecto. El orchestrator interactivo debe pasar `force_confirm: true`
  después de pedir confirmación al usuario.
- La precisión de las bounding boxes depende del modelo de visión. `gemini-2.0-flash`
  da coordenadas razonables pero no píxel-perfect; siempre observar antes de actuar.
